\subsection{System Architecture and Modeling Framework}

Benchmarking post-quantum algorithms consistently across different IoT hardware tiers requires a strictly controlled testing framework. Figure \ref{fig:framework} illustrates the step-by-step isolation method our profiling tool uses to capture these metrics.

\begin{figure*}[htbp]
\centering
\includegraphics[width=0.85\textwidth]{fig1_framework}
\caption{PQC IoT Benchmarking Framework Architecture}
\label{fig:framework}
\end{figure*}

An isolated measurement pipeline is required to ensure accurate profiling. Background OS interference and dynamic linker overheads are stripped away by combining \texttt{cgroups} with kernel-level \texttt{getrusage()} calls on the \texttt{liboqs} binaries. Ultimately, this deterministic environment allows the actual bare-metal ARM Cortex performance to be reliably derived from the high-level Python/C execution times.
