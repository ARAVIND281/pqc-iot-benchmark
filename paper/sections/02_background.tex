\section{Background and Mathematical Foundations}
\label{sec:background}
%-----------------------------------------------------------------------

To comprehend the distinct performance divergences intrinsic to post-quantum encryption and authentication, we survey the core mathematical assumptions underpinning the NIST algorithm portfolio.

\subsection{Mathematical Foundations of PQC}

\subsubsection{Lattice-Based Cryptography (Module-LWE)}
The security of lattice-based mechanisms anchors on Learning With Errors (LWE) and Module-LWE \cite{regev2009lattices}. Given a public matrix $\mathbf{A} \in \mathbb{Z}_q^{k \times k}$ and a secret vector $\mathbf{s}$, an adversary is challenged to find $\mathbf{s}$ given $\mathbf{t} = \mathbf{A}\mathbf{s} + \mathbf{e} \pmod q$, where $\mathbf{e}$ is a small error polynomial. 

\textbf{ML-KEM (FIPS 203)} operates over the ring $R_q = \mathbb{Z}_q[X]/(X^{256} + 1)$ with $q = 3329$. It provides efficient key encapsulation but demands substantial register availability for polynomial multiplications. 

\textbf{ML-DSA (FIPS 204)} operates symmetrically to ML-KEM but adopts the ``Fiat-Shamir with Aborts'' framework. During signing, to prevent secret key leakage, the signature is rejected and restarted if coefficients exceed predefined bounds (rejection sampling). This non-deterministic loop drains energy reserves rapidly on Class 1 devices lacking large CPU caches, while keys exceed 2.5 KB, demanding fragmented transmission.

\subsubsection{\ntru{} Lattices and FN-DSA}
\textbf{FN-DSA (Draft FIPS 206)}\footnote{NIST FIPS 206 remains a draft standard pending finalization of specifications.} solves the signature size dilemma of ML-DSA (reducing sizes to 666 bytes for 128-bit security) by utilizing dense \ntru{} lattices \cite{hoffstein1998ntru}. However, generating signatures requires continuous Gaussian sampling over high-precision floating-point arithmetic. 

\subsubsection{Hash-Based Signatures (SLH-DSA)}
\textbf{SLH-DSA (FIPS 205)} secures digital signatures upon proven hash function properties (e.g., SHA-256), circumventing unproven algebraic constructs \cite{bernstein2019sphincs}. It constructs a hyper-tree combining Few-Time Signatures (FORS) and eXtended Merkle Signature Schemes (XMSS). While mathematically conservative, the resulting signature encapsulates the entire authentication path. An SLH-DSA-128s signature consumes roughly 8 KB, rendering it hostile to Class 1 UDP/CoAP constrained endpoints.

\subsubsection{Code-Based Cryptography (HQC)}
\textbf{HQC} derives security from the syndrome decoding problem \cite{nist2025hqc}. It bypasses NTT convolutions entirely, opting for simpler vector additions over finite fields GF(2), offering ultra-low power execution despite large public keys.

\subsection{Defined Deployment Targets: RFC 7228}

To maintain rigid boundaries reflective of genuine embedded hardware, we evaluate using the strict taxonomy formalized by the IETF's RFC 7228 \cite{bormann2014coap}:

\begin{itemize}
 \item \textbf{Class 0 (C0):} Devices exhibiting $<10$ KB RAM and $<100$ KB Flash storage. Commonly bare-metal sensory motes devoid of standard TCP/IP networking, necessitating proxy gateways.
 \item \textbf{Class 1 (C1):} Devices exhibiting $\approx 10-50$ KB RAM and $\approx 100-250$ KB Flash. Suitable for lightweight Datagram TLS (DTLS) and UDP implementations (e.g., nRF52840 nodes).
 \item \textbf{Class 2 (C2):} Devices exhibiting $\approx 50-250$ KB RAM and $\approx 250$ KB - 1 MB Flash. Readily capable of implementing nearly full TCP/IPv6 stacks (e.g., robust ESP32 environments).
\end{itemize}

\subsection{Related Work}

Over recent years, embedded PQC benchmarking has expanded. The pivotal \textbf{pqm4 project} \cite{kannwischer2019pqm4} successfully ported highly optimized ARM Cortex-M4 assembly targets to the cryptographic community, serving as the definitive baseline instruction set reference.

More recently, \textbf{Fitzgibbon and Ottaviani (2024)} \cite{fitzgibbon2024benchmarking} extended multi-platform evaluations across Raspberry Pi micro-architectures, concluding that while KEM deployments are feasible, generic digital signature scaling remains significant for memory pools. Concurrently, \textbf{Lopez et al. (2025)} \cite{lopez2025post} profiled PQC primitives against active WiFi configurations, isolating garbage collection degradation occurring exclusively during concurrent radio activation. At SECRYPT 2025, \textbf{Silva et al.} \cite{secrypt2025esp32} conducted deeply compartmentalized tests upon the immensely popular ESP32 SoC targeting FreeRTOS environments, observing the physical limitations of hardware acceleration absences impacting \falcon{} FFT floating-point sampling bounds. Furthermore, \textbf{Demir et al. (2025)} \cite{demir2025power} conducted advanced power trace correlations demonstrating that naive constant-time modifications dramatically escalate power consumption.

Despite these vital benchmarks forming subsets of the problem geometry, literature is noticeably devoid of unified Class-spanning data grids identifying the precise feasibility cutoff thresholds for the aggregate suite of algorithms, particularly when normalized under multi-attribute penalty considerations like TOPSIS.
