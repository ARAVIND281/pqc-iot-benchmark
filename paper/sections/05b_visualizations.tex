\subsection{Variance and Execution Consistency}

IoT environmental constraints frequently induce substantial stochastic operating system jitter. Figure 10 maps the execution timing variances using box plots.

\begin{figure*}[htbp]
\centering
\includegraphics[width=\textwidth]{fig6_boxplot}
\caption{Boxplot distribution highlighting significant jitter anomalies distinctly present in memory-heavy instantiations.}
\label{fig:boxplot}
\end{figure*}

This paper found that memory-heavy algorithms produce wider interquartile ranges (IQR). These heavy memory demands cause long-tail latency events. Specifically, the CPU stalls when it hits flash-memory page boundaries or triggers garbage collection. These unpredictable delays cause industrial sensors to fail their strict Real-Time Operating System (RTOS) deadlines.

\subsection{Correlation Matrices and Radar Fingerprints}

To analytically prove the independence of our target variables (Energy vs Memory vs Time), Pearson correlation coefficients were generated (Figure \ref{fig:correlation}).

\begin{figure}[htbp]

However, Key Size possesses a dramatically lower $+0.42$ linkage to CPU Energy, proving that radio transmissions and calculation times interact as independent resource vectors.

\subsection{Security Level vs Resource Pareto Frontier}

The quintessential dilemma facing IoT network architects globally is identifying the exact mathematical trade-off curve between NIST Category scaling (Level 1 $\rightarrow$ 5) versus physical battery drainage.

\begin{figure*}[htbp]
\centering
\includegraphics[width=\textwidth]{fig14_pareto}
\caption{Pareto Frontier analyzing global Resource Drain multipliers linearly versus NIST security gradations identically.}
\label{fig:pareto}
\end{figure*}

Figure \ref{fig:pareto} unambiguously highlights the optimal deployment boundary. Algorithms falling along the Pareto optimal curve (\kyber{}-512, \falcon{}-512) maximize cryptographic resistance while utilizing physically minimal resources globally. Algorithms pushed violently rightward (e.g., \sphincs{}-256) offer theoretical absolute security while requiring physically untenable resource footprints triggering operational failures.
