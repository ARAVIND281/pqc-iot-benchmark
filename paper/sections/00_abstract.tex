\begin{abstract}
Despite the National Institute of Standards and Technology (NIST) finalizing the first set of post-quantum cryptographic (PQC) standards in August 2024, a critical gap remains: there is no systematic cross-algorithm benchmarking across all three RFC 7228 Internet of Things (IoT) constrained device classes. The imminent threat of cryptographically relevant quantum computers executing Shor's algorithm, combined with the ``harvest now, decrypt later'' attack model, necessitates the immediate transition of vulnerable IoT infrastructure. This paper presents a comprehensive empirical analysis of five primary PQC algorithms---\kyber{}, \dilithium{}, \falcon{}, \sphincs{}, and \ntru{}---across 14 parameter sets spanning Class 0, Class 1, and Class 2 IoT constraints. Through over 45,000 rigorous empirical approximations aligned with bare-metal instruction limits, we evaluate execution time, memory footprint, energy consumption, and communication overhead. We introduce an expanded energy estimation model accounting for both cryptographic processor cycles and radio transmission latency, alongside a multi-criteria TOPSIS decision framework to generate actionable algorithm selection guidelines. Our statistical modeling, underpinned by ANOVA and Tukey HSD significance testing ($p<0.001$), demonstrates that while Class 0 devices are infeasible for direct PQC implementation without designated gateway delegation architectures, ML-KEM-512 and ML-DSA-44 provide optimal Pareto-efficient trade-offs for Class 1 and Class 2 networks respectively. We conclude by projecting simulated TLS 1.3 handshake latencies, assessing hybrid classical-quantum transition overheads, and outlining a migration roadmap. To accelerate reproducible IoT security research, we open-source our comprehensive benchmarking toolchain and multi-dimensional datasets.
\end{abstract}
